3. KRITERIA AHP DAN BOBOT
================================================================================

Volume Penjualan (63.7%) - Benefit
  Normalisasi: Min-Max Scaling
  Formula: (X - Min) / (Max - Min)
  Tujuan: Mencari barang fast-moving

Stok Minimum (25.8%) - Cost/Urgency
  Normalisasi: Rasio Stok
  Formula: Stok Saat Ini / Stok Minimum
  Tujuan: Prioritas stok mendekati/di bawah ambang batas

Harga Satuan (10.5%) - Benefit
  Normalisasi: Min-Max Scaling
  Formula: Harga / Harga Maksimum
  Tujuan: Optimalisasi perputaran modal

Skor Akhir:
  Final Score = (Sales × 0.637) + (Stock × 0.258) + (Price × 0.105)
  Range: 0-1 (0% - 100%)

Kategori Prioritas:
  Sangat Tinggi (≥0.7): Restock URGENT
  Tinggi (0.5-0.69): Restock dalam waktu dekat
  Sedang (0.3-0.49): Restock normal
  Rendah (<0.3): Monitor saja

================================================================================
4. PENANGANAN PERBEDAAN SATUAN
================================================================================

Sistem menangani satuan berbeda TANPA konversi absolut:

Volume Penjualan: Dinormalisasi relatif terhadap nilai maksimum
Stok Minimum: Rasio dihitung per produk tanpa konversi
Harga Satuan: Nilai absolut, tidak terpengaruh satuan fisik

Contoh:
  Produk A: 50 Kg, Min 20 Kg, Rasio = 2.5
  Produk B: 100 Pcs, Min 50 Pcs, Rasio = 2.0
  
  Kedua produk dinilai FAIR meskipun satuan berbeda
  karena perhitungan berbasis rasio, bukan nilai absolut.

5. METODE AKSES
================================================================================

Akses Menu:
  1. Login sebagai Admin
  2. Klik menu "SPK Prioritas (AHP)" di sidebar
  3. Halaman akan menampilkan analisis lengkap

Filter Kategori:
  1. Pilih kategori dari dropdown
  2. Klik tombol "Terapkan Filter"
  3. Atau klik "Reset" untuk kembali ke semua kategori

Cetak Laporan:
  1. Pilih kategori (opsional)
  2. Klik tombol "Cetak Laporan"
  3. Jendela print akan terbuka
  4. Klik "CETAK LAPORAN" atau Ctrl+P untuk print

API Endpoint:
  GET /admin/api/ahp-data
  GET /admin/api/ahp-data?category_id=5&limit=20

================================================================================
6. PERSYARATAN DATA PRODUK
================================================================================

Setiap produk HARUS memiliki:
  ✓ min_stock (integer) - Stok minimum
  ✓ sales_frequency (integer) - Frekuensi penjualan
  ✓ unit (string) - Satuan (Pcs, Kg, Liter, Dus, dll)
  ✓ price (integer) - Harga satuan
  ✓ category_id (integer) - ID kategori
  ✓ stock (integer) - Stok saat ini

Jika ada field yang kosong, produk tidak akan dianalisis.

================================================================================
7. TROUBLESHOOTING
================================================================================

Data Produk Tidak Muncul:
  Pastikan setiap produk memiliki min_stock, sales_frequency, unit, price

Skor AHP Semua Sama:
  Pastikan data produk memiliki variasi nilai untuk ketiga kriteria

Bar Chart Tidak Tampil:
  Cek koneksi internet (CDN Chart.js) atau buka browser console

Filter Kategori Tidak Bekerja:
  Pastikan kategori memiliki produk atau refresh halaman

Laporan Cetak Tidak Rapi:
  Gunakan Chrome/Firefox, atur margin ke Default, centang "Background Graphics"
